Alexya provides an easy way to connect to a database and execute queries. Alexya also provides a fast and easy to use ORM class out of the box.

The class \Alexya\Database\Connection establishes a connection with the database and provides methods for executing queries. When Alexya is bootstrapped it automatically instances the $Database object if specified in the configuration file.

Method summary:

Name Parameters Return value Description
_setOptions Array $options: Database settings array boolean This method is called when the controller is instanced, instead of overriding the constructor you can override this method.
isRouteable string $method: Method name boolean Checks that $options is a valid array containing options and sets internal properties for connection, returns whether it succeeded or not.
close void Closes current connection.
connect void Connect to the server, throws a \Alexya\Exceptions\Database\ConnectionFailed exception f couldn't connect to host.
isClosed boolean Checks whether connection is closed or not.
getConnection PDO Returns the PDO object with current connection.
query string $query: Query to execute Array Executes a raw query and returns result as an array.
Properties summary:
Name Type Visibility Description
_host string Private Database server.
_port int Private Server port.
_type string Private Server type.
_username string Private User name.
_password string Private Password.
_database string Private Database name.
_options array Private Alternative PDO options.
_connection PDO Private PDO connection.
lastQuery string Public Last executed query.

Using the \Alexya\Database\Connection class is fair easy: 

$users = $Database->query("SELECT * FROM `users`");

foreach($users as $user) {
    echo "Username: ". $user["name"];
    echo "Email: ". $user["email"];
}

$user = $Database->query("SELECT * FROM `users` WHERE `id`=1");

echo "Username: ". $user["name"];
echo "Email: ". $user["email"];
Sometimes you'll need to handle more than one database connection. If so you can instance a new connection object by sending an array with connection settings (same as the array on config/database.php) as parameter:
<?php
/**
 * Load Alexya's core
 */
require_once("bootstrap.php");

$Database = new \Alexya\Database\Connection([
    "host"     => "localhost",
    "port"     => 3306,
    "type"     => "mysql",
    "username" => "root",
    "password" => "",
    "database" => "alexya",
    "options"  => []
]);
If you want to take a total control over \Alexya\Database\Connection class you can use the method \Alexya\Database\Connection::getConnection that retruns the current PDO object with the database connection.

When an error happens the method \Alexya\Database\Connection::getError returns an array with the latest error:
<?php
/**
 * Load Alexya's core
 */
require_once("bootstrap.php");

$users = $Database->query("SELECT FROM `users`");
if(empty($users)) {
    $error = $Database->getError();

    echo "An error happened!\n". $error[2] .", error code: ". $error[1];
}
See PHP: PDO::errorInfo for more info about the return array.
