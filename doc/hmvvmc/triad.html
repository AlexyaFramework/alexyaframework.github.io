The class \Alexya\Triad links the Model, View, Presenter and Controller in a single class and provides an easy way for implementing the HMV(VM)C design pattern.

Method summary:

Name Parameters Return value Description
__get string $name: Name of the object to get mixed Lazy instances and returns any of the MV(VM)C components (or false if anything failed).
__set string $name: Name of the component to set
object $value: Component's object 		void Overrides the instance of $name with $value.
addChild string $name: Name to identify the triad
\Alexya\Triad $triad: Triad object 		void Adds a child triad.
getChild string $name: Name to identify the triad \Alexya\Triad Returns a child from the array.
setModelName string $name: Model name Sets Model's name, use this if the Model of the triad couldn't be instanced.
setControllerName string $name: Controller name Sets Controller's name, use this if the Controller of the triad couldn't be instanced.

The constructor accepts as parameter a string and a \Alexya\Http\Request object that is the name of the triad and the request for that triad.

$triad = new \Alexya\Triad("User", \Alexya\Http\Request::main);

$response = $triad->Controller->index();
$response->send();
When working with HMV(VM)C is good practice to keep the parent triads and the children triads in separated folders. To do this instance an object of type \Alexya\Page for a parent triad or an object of type \Alexya\Module for a child triad, the directories where the MV(VM)C components are saved are Application/Page and Application/Module respectively, if you don't like this names you can extend the \Alexya\Triad class and specify a name for the directory:
class ChildTriad extends \Alexya\Triad
{
    public $suffix = "\\Application\\Child\\";
}


$triad = new ChildTriad("User", \Alexya\Http\Request::main);

$response = $triad->Controller->index();
$response->send();
The components of this kind of triads are in the directory Application/Child.
