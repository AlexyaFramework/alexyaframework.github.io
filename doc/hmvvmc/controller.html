The controller represents the C in MV(VM)C and it contains the logic of the triad. The class \Alexya\Controller is the base class for all controllers of your application.

Method summary:

Name Parameters Return value Description
onInstance void This method is called when the controller is instanced, instead of overriding the constructor you can override this method.
isRouteable string $method: Method name boolean Checks whether $method can be accessed through an URL call.
For example:
If the controller 
\Application\Controller\User
can be accessed through the URL 
http://localhost/User
all the methods of it can be called like 
http://localhost/User/deleteAllUsers

If the method 
\Application\Controller\User::deleteAllUsers
can be routed, this method will return true, if not will return false
index \Alexya\Http\Response The default method to execute when no other action has been called.
Properties summary:
Name Type Visibility Description
noRouteable Array Public An array containing the methods that can't be executed through an URL call.
request \Alexya\Http\Request Public The request that was sent to instance the controller.
triad \Alexya\Triad Public A reference to the \Alexya\Triad object that instanced the controller.

The methods on the controllor that can be routed must return an object of type \Alexya\Http\Response that will contain the response to send.
For example, consider having this controller that can be accessed through the URL http://localhost/User: 

<?php
namespace Application\Controller;

use \Alexya\Controller;
use \Alexya\Http\Response;

/**
 * Controlller for the 'User' page
 *
 * @author Manulaiko <manulaiko@gmail.com>
 *
 * @package \Application\Controller
 */
class User extends Controller
{
    /**
     * Set non routeable methods
     */
    public function onInstance()
    {
        $this->noRouteable[] = "deleteAllUsers";
    }

    /**
     * Index action
     *
     * The default action that will be executed when
     * router couldn't find any other action
     *
     * @return \Alexya\Http\Response Response object
     */
    public function index() : Response
    {
        $response = new Response();

        $response->headers->add("Content-Type", "text/html");
        $response->body = $this->triad->Presenter->render();

        return $response;
    }

    /**
     * Updates user data
     *
     * @return \Alexya\Http\Response Response object
     */
    public function Update() : Response
    {
        $response = new Response();

        //Do things

        return $response;
    }

    /**
     * Deletes all users
     */
    public function deleteAllUsers()
    {
        //Do bad things with the database
    }
}
The onInstance method adds a non routeable method to the array: deleteAllUsers.
The Update method is routeable and returns an object of type \Alexya\Http\Response, meaning that it can be called trough the URL. The method \Application\Controller\User::index calls the render method of the presenter of its triad. If the router couldn't find any method to execute it will call this method and send the response.
If the request URL is http://localhost/Update, the method \Application\Controller\User::Update will be called and the response object it returns will be sent to the server.
If the request URL is http://localhost/deleteAllUsers, the method \Application\Controller\User::index will be called because the method \Application\Controller\User::deleteAllUsers can't be routed.
