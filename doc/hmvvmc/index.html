Alexya supports the HMV(VM)C design pattern, although you don't need to use it it's recommended to use it.

The MVC design pattern lets you divide your application in 3 differents in 3 different components: The model, the view and the controller. Each of them in a separated file.

One of the inconvenients of MVC is that the controller handles all the logic, incluiding the logic for the view.
To avoid the overloading of code in the controller the design pattern MVVM adds another component called ViewModel (or as it's called in Alexya, a Presenter) that contains the logic for the view, so instead of having Model, View and Controller we have a Model, View and ViewModel.
The inconvienent of this is that it deletes the Controller from the design pattern. Affortunaly we can mix both design patterns to create the MV(VM)C which contains Model, View, ViewModel and Controller.

Finally the HMVC design pattern provides a hierarchical MVC, meanning each MVC (or Triad, as it's called in Alexya) can have a parent triad and one or more children triads.

Alexya not only supports all of those designs patterns but lets you mix all of them in a single HMV(VM)C design pattern for extra fun effect.
