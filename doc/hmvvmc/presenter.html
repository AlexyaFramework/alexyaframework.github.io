The presenter (ViewModel) represents the VM in MV(VM)C and it contains the logic to display the view. The class \Alexya\Presenter is the base class for all presenters of your application.

Method summary:

Name Parameters Return value Description
onInstance void This method is called when the presenter is instanced, instead of overriding the constructor you can override this method.
render string Renders the view and returns its output.
__toString string Alias for render.
__set string $key: Key in $_data array
mixed $value: Value for $key 		string Alias for set.
__get string $key: Key in $_data array string Alias for get.
__isset string $key: Key in $_data array string Checks whetheer a variable is set in the array or not
__unset string $key: Key in $_data array string Unsets a variable from the array.
set string $key: Key in $_data array
mixed $value: Value for $key 		string Adds a variable to the array.
get string $key: Key in $_data array string Retrieves a variable from the array.
Properties summary:
Name Type Visibility Description
request \Alexya\Http\Request Public The request that was sent to instance the presenter.
triad \Alexya\Triad Public A reference to the \Alexya\Triad object that instanced the presenter.

Example for the User presenter:

<?php
namespace Application\Presenter;

use \Alexya\Presenter;

/**
 * Presenter for the 'User' page
 *
 * @author Manulaiko <manulaiko@gmail.com>
 *
 * @package \Application\Controller
 */
class User extends Presenter
{
    /**
     * Set variables
     */
    public function onInstance()
    {
        $model = $this->triad->Model;

        $this->set("name", $model->name); //Using method `set`
        $this->set("password", $model->password); //Using method `set`
        
        $this->email = $model->email; //Using magic method `__set`
    }
}
